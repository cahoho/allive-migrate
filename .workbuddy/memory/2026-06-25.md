## 排行榜/数据库问题诊断与修复

### 问题：游戏结束后数据不显示在排行榜中

### 根因诊断
通过直接测试 Supabase Edge Functions 发现：
1. `get-leaderboard` 正常 (HTTP 200)，但返回空数据 → players 和 game_sessions 表均为空
2. `submit-score` 正常 (HTTP 403，因为传入无效 token，说明函数可用)
3. **`start-session` 返回 HTTP 500** (`{"error":"Failed to create session"}`) → 核心问题

### 根本原因：seed INT32 溢出
- `start-session/index.ts` 中 `generateSeed()` 使用 `crypto.getRandomValues(new Uint32Array(1))[0]`
- 返回无符号 32 位值 (0 ~ 4294967295)，但 PostgreSQL `seed INTEGER` 是有符号 32 位 (max 2147483647)
- ~50% 的随机种子超过上限，导致 INSERT 失败 → 500 错误
- sessionToken 为 null → GameOverModal 静默跳过分数提交 → 排行榜永为空

### 修复内容
1. **【核心】supabase/functions/start-session/index.ts**: `generateSeed()` 增加 `& 0x7FFFFFFF` 确保 seed 在 INT32 范围内
2. **src/components/GameOverModal.vue**: 
   - sessionToken 为 null 时显示明确错误提示（之前静默跳过）
   - 网络错误时显示重试按钮
   - catch 块显示详细错误信息
3. **src/components/LeaderboardModal.vue**: 静默 catch 改为显示错误信息
4. **setup-supabase.sh**: 修正 PROJECT_REF 从 `xqjonnyygbikbcansebam` 改为 `pzufflfqbeubpdoovawp`
5. **GameOverModal.vue restart()**: startSession 失败时清除旧 token，避免 TOKEN_USED 错误

## updateNickname "先更新再无匹配则 upsert" 逻辑修复 (20:58)
- 根因：Supabase 的 `.update().eq("device_id", id)` 在没有匹配行时返回 `error: null`（不是 error），导致 `if (error)` 判断不成立，upsert 兜底从不触发。首次游戏的玩家从未被写入 players 表

## 排行榜得分/时间为 0 问题修复 (22:38)

### 问题
每个玩家游戏结束后可以上榜，但得分和时间始终为 0，而网页界面显示的是正确的分数和时间。

### 根因诊断
`GameOverModal.vue` 使用 `onMounted` 生命周期钩子触发 `trySubmitScore()`。但组件在 `appPhase === 'playing'` 时就已经挂载（而非 `gameOver === true` 时），导致：

1. **教程模式**：`onMounted` 时 `gameStarted=false` → `submitDone.value = true`，之后游戏真正结束时 `submitDone=true` 导致提交被跳过
2. **非教程模式**：`onMounted` 时游戏刚启动（score=0, survivalTime=0）→ 提交了 0/0 数据并设置 `submitDone=true`，后续无法覆盖

### 修复内容
`src/components/GameOverModal.vue`:
- 移除 `onMounted` 钩子
- 新增 `watch` 监听 `gameStore.state.gameOver`
- `gameOver=true` 时触发 `trySubmitScore()`
- `gameOver=false` 时（新一局开始）重置 `submitDone`、`submitError`、`isNetworkError`、`isSubmitting`、`myRank`
- `trySubmitScore()` 顶部增加防御性检查：`if (!gameStore.state.gameOver) return`

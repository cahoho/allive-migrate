> **文档版本**: v1.0 | 2026-06-25

> **赛题**: 赛题一「小红花游戏」— 生物多样性与环境保护方向

## 第一部分：整体风格与设计规范

### 1.1 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#0D1B14`（深墨绿） | 所有内容页统一深色背景，营造自然沉浸感 |
| 辅背景 | `#14241E`（松烟绿） | 卡片/区块背景，比主背景略亮一层 |
| 主强调色 | `#4ADE80`（翠绿） | 标题高亮、关键数据、成功状态 |
| 次强调色 | `#F59E0B`（暖金/琥珀） | 数据节点、阶段标识、亮点高亮 |
| 辅助色 | `#38BDF8`（天蓝） | 物种生物相关、迁徙路线、水系元素 |
| 警示色 | `#EF4444`（警示红） | 灭绝警告、人类威胁、生态危机 |
| 正文文字 | `#D1D5DB`（浅灰白） | 主要正文，确保深色背景上的可读性 |
| 次要文字 | `#9CA3AF`（中灰） | 辅助说明、标签、脚注 |
| 装饰线 | `#1E3A2F`（暗绿线） | 分隔线、边框 |

### 1.2 字体规范

- **标题字体**: 思源黑体 Bold / Noto Sans SC Bold（36-48pt）
- **正文字体**: 思源黑体 Regular / Noto Sans SC Regular（18-24pt）
- **数据/代码字体**: JetBrains Mono / Source Code Pro（16-18pt）
- **英文副标题**: Inter / SF Pro Display（与中文标题搭配）

### 1.3 版式规范

- **比例**: 16:9（1920×1080）
- **安全区**: 上下左右各保留 80px 边距
- **标题位置**: 每页左上角，36pt + 英文副标题 16pt（透明度0.5）
- **卡片风格**: 圆角 12px，半透明背景（`rgba(20,36,30,0.85)`）+ 1px 暗绿边框
- **图标**: 使用 Lucide Icons 或物种 SVG 图标，线条风格
- **数据展示**: 优先使用卡片 + 大数字，减少长段落文字

### 1.4 通用设计原则

1. **每页一个核心信息点**：不要让评审在一页内读超过3个要点
2. **数据可视化优先**：能用图表就不用文字，能数字就不用描述
3. **深色底 + 高对比**：适合投影环境，保证后排可见
4. **自然元素贯穿**：绿色渐变装饰线、物种剪影水印
5. **动画克制**：页面内过渡用 fade-in，避免花哨切换

### P10 · 物种多样性与失败条件

**标题**: 什么会让你输？——多样性塌缩

**内容要素**:
- **左侧**: 灭绝机制图
  ```
  同一物种失败3次 → 该物种"灭绝" → 从候选池永久移除
  ↓
  多样性 = 存活物种数 / 已解锁物种数 × 100%
  ↓
  多样性 < 33.3% → 游戏结束
  ```
- **右侧**: 设计意图卡
  - "失败的不是'一条路线'，而是一个物种的未来"
  - "多样性的塌缩比任何一次单独失败都更致命"
  - "这与现实生态学原理一致：生态系统的韧性取决于多样性"
- **底部**: 对标评审——游戏平衡性 + 叙事深度

**设计意图**: 展示失败条件的叙事价值，不只是"死了就重来"

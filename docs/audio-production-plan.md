## 0. 需求分析摘要

### 0.1 游戏核心概念

《Allive Migration · 众生迁徙》是一款动态生态网络游戏。玩家在 5–10 分钟内保护生态通道，引导迁徙物种从起点抵达终点。音频不是"背景音乐"，而是**生态网络的听觉投影**——每个机制变化都必须在声音里有可被玩家感知的对应物。

### 0.2 五大设计支柱（不可妥协）

| # | 支柱 | 音频规范要求 |
|---|------|-------------|
| P1 | **音乐即机制** | 解锁/失败/事件必须引发可听变化，玩家应能"听到自己做了什么" |
| P2 | **层叠式混音** | L1 核心层 + L2 物种层 + L3 状态层，三者独立可控 |
| P3 | **协和优先** | 所有物种乐器共享 D 大调调性中心，和声永远不冲突 |
| P4 | **节俭的反馈** | 5–10 分钟单局，音效不能比玩法更累 |
| P5 | **可沉默运行** | 静音下游戏必须完整运行，音频引擎恢复时平滑续接 |

### 0.3 三层混音骨架

```
L3 · 状态层 (State Layer)   — 灾害 / 阶段 / 危机     ← 最响、变化最慢
L2 · 物种层 (Species Layer) — 已解锁物种的乐器轨道    ← 中响、随解锁增减
L1 · 核心层 (Core Layer)    — 主旋律 + 和声底色 + 律动  ← 全程在场
────────────────────────────────────────────────
FX · 一次性反馈音 (One-Shot) — 拖线、成功、失败       ← 短促、立体声定位
```

## 2. 旋律风格指南

### 2.1 节奏 DNA

| 参数 | 值 | 约束 |
|------|-----|------|
| **基准 BPM** | **78** | 范围 72–84 |
| **拍号** | 4/4 | 嵌入 3+3+2 五拍感 |
| **底鼓规则** | **永不出现标准四拍底鼓** | 重音落第 1 拍和第 3 拍的 **&** |
| **速度感来源** | 铃/拨弦的"叮咚"代替鼓点 | 轻而有方向 |
| **类比参考** | Pixar 短片轻盈感 × Sigur Rós 呼吸感 | 体感参考，非版权对照 |

### 2.2 和声逻辑

| 参数 | 值 | 技巧 |
|------|-----|------|
| **调性中心** | **D 大调** (root MIDI = 62) | E♭ 大调为危机副调 |
| **核心进行** | `I → vi → IV → V` (D → Bm → G → A) | 所有变体必须落在四和弦子集 |
| **禁止** | 减七和弦、半音下行、属七→主"解决" | 太俗套 |
| **特色手法** | sus2/sus4 挂留和弦 | 制造"未完成"感 |
| **借用和弦** | 从 d 小调借 Gm(G→iv)、B♭(B→VI) | 表达"被威胁"，不切调 |

### 2.3 主题隐喻：飞行轨迹

- 旋律以 **D4 (MIDI 62)** 开始，上行到 **A5 (MIDI 81)** 附近
- 用 **3 度音程跳进** 而非 2 度级进——鸟的拍翅节奏
- 每个乐句不在主音结束，在三音或五音悬停
- 永远不会"到达"稳定终止式——永远在迁徙中

### 2.4 三个动机单元

| 动机 | 名称 | 时长 | 音程特征 | 用法 |
|------|------|------|---------|------|
| **M1** | 起飞 | 2 小节 (6.2s) | 上行 3 度跳进 ×3 | 段落开头、解锁瞬间 |
| **M2** | 滑行 | 4 小节 (12.3s) | 持续颤音 + 弧线 | 主旋律主体 |
| **M3** | 盘旋 | 2 小节 (6.2s) | 装饰音环绕(主音) | 任务开始、状态切换 |

**关键规则**: 任一时刻只能有 1 个动机在听感上"主导"，其余动机以 1/4 音量做伴奏。

### 2.5 五种发展方式（操作原语）

| 发展方式 | BGM 变化 | 技能实现提示 |
|---------|---------|-------------|
| **重复** | 直接重复 M2 | 循环播放同一段 |
| **移调** | M2 整体上/下移 2 度 | 调整 root_midi ±2 |
| **装饰** | M2 长音叠加倚音、回音 | melody_style: `arpeggio` 或 `dense` |
| **切断** | M2 被静音(被风吹断) | 音量→0, 保持静默 1.5s 后 M3 低八度复入 |
| **倒影** | M1 音程反转(上行变下行) | 旋律方向反转，BPM 减速 |

## 3. 乐器配置映射

### 3.1 L1 核心层乐器（常驻）

|----------|-------------|------|------|----------|
| 木吉他(尼龙弦) | `pluck` | 250 Hz–2 kHz | 主旋律载体，指弹分解+单音旋律 | 60–81 |
| 颤音琴 | `bell` | 800 Hz–4 kHz | 和声铺底+旋律呼应，4Hz 马达颤音 | 66–93 |
| 大提琴 | `string` | 80–400 Hz | 低音根音+情绪厚度，长弓持续音 | 38–62 |

### 3.2 L2 物种~乐器完整映射

| 物种 ID | 物种名 | GDD 乐器 | 技能映射 | 频段区间 | 音乐角色 |
|---------|--------|---------|---------|---------|---------|
| `bird` | 雁鸭候鸟 | 长笛 | `flute` | 250 Hz–3 kHz | 主旋律领奏，比木吉他高 8 度 |
| `butterfly` | 帝王蝶 | 竖琴 | `pluck` (琶音) | 200 Hz–4 kHz | 装饰性琶音，M3 盘旋装饰音 |
| `bar_goose` | 斑头雁 | 短笛 | `flute` (高音) | 2 kHz–6 kHz | 高频闪烁，M1 起飞叠加 3 个高音短音 |
| `salmon` | 鲑鱼 | 大提琴独奏 | `string` | 80–400 Hz | 低音根音，M2 滑行低音镜像(下 8 度) |
| `herd` | 角马兽群 | 中提琴 | `string` (中音) | 150–800 Hz | 中频填充，M2 中段"踏步感"长音 |
| `eel` | 美洲鳗 | 低音单簧管 | `bass` (暗色) | 120 Hz–1.5 kHz | 暗色连接，M2 滑行"水下"变体+滑音 |

### 3.3 频段轮盘分配

```
L1 (60-200 Hz)     → string (大提琴,独占)
L2 (200-400 Hz)    → string (大提琴 + 中提琴)
L3 (400-800 Hz)    → pluck (木吉他低音)
L4 (800 Hz–1.6 kHz) → pluck (木吉他主音) / flute
L5 (1.6–3.2 kHz)   → flute (长笛) / pluck (班卓)
L6 (3.2–5 kHz)     → bell (颤音琴主体)
L7 (5–7 kHz)       → bell (钢片琴) / flute (短笛)
L8 (7 kHz+)        → bell (泛音, 低能量)
```

### 3.4 禁止使用的音色

| 禁止项 | 原因 |
|--------|------|
| 失真电吉他 / 重型合成器 pad | 与其他层互相撕扯 |
| 标准电鼓 / 808 / 909 | 与"轻而有方向"的律动矛盾 |
| 失真贝斯 | 破坏低频纯洁性 |
| 响板 / 手鼓等攻击性打击乐 | 与脆弱感冲突 |
| 大编制弦乐群 (>12 人) | 留给大提琴 solo 替代，避免堆砌 |

## 4. 技术规格标准

### 4.1 交付格式

| 维度 | 规范 |
|------|------|
| **母带格式** | WAV 44.1 kHz 16-bit 立体声 |
| **运行时压缩** | OGG/Opus (Web Audio 性能优化) |
| **循环音频** | 8 的整数倍小节，0.5 秒淡入淡出 + 采样级无缝 |
| **一次性音频** | 5ms 头淡入避免咔哒声 |
| **文件命名** | `a_<layer>_<role>_<variant>.wav` |
| **元数据** | JSON: `{bpm, key, duration_s, rootFreq, fadeIn, fadeOut, layer, role}` |
| **总量目标** | < 25 MB (母带) |

### 4.2 运行时性能约束

| 约束 | 限制 | 备注 |
|------|------|------|
| 并发 AudioBufferSourceNode | ≤ 32 | 浏览器硬限 32–64 |
| 一次性反馈音最大并发 | ≤ 8 | 同源混音防爆音 |
| 总运行时内存 | < 40 MB | 含解码后 PCM |
| 首屏加载 | < 5 MB | 仅 L1 核心 + FX，其余懒加载 |
| 自动播放激活 | 首次 `pointerdown` 触发 AudioContext | 符合浏览器规范 |

### 4.3 音频状态机（11 状态）

```
BOOT → IDLE → PLAY_BASE → PLAY_LAYERED → DISASTER → RECOVER
                                    ↓                      ↓
                              CRISIS → ENDGAME
                                    
PAUSE / SOFT_MUTE / TEARDOWN (全局可切换)
```

## 5. 互动机制实现细节

### 5.1 八个互动机制速查

| 机制 | 触发条件 | 音乐变化 | 过渡时间 | 音量变化 |
|------|---------|---------|---------|---------|
| M-01 物种解锁 | `tryUnlockSpeciesForStage` 成功 | 叠加物种乐器轨道，淡入至 0.7 | 1.8s | 0→0.7 |
| M-02 灾害叠加 | 灾害进入 active | L3 状态层接管，L1 退至 1/4 | 1.5–4.0s | L3:0→0.6, L1:1.0→0.25 |
| M-03 阶段跃迁 | 累计成功达标 | M1+M2 上移 2 度，D→E 大调 | 4 小节 | 不变 |
| M-04 失败失真 | 失败+1 或物种灭绝 | 物种轨道 0.6s 淡出 + 2s bit-crush 染色 | 0.6s | 0.7→0 |
| M-05 健康颤音 | 节点 health < 40 | 颤音琴马达颤音 4Hz→7Hz | 即时 | 不变 |
| M-06 拖拽预听 | 拖线经过节点 | 乐器点状提示音 (60ms, velocity 0.4) | 即时 | 0.4 |
| M-07 终局减速 | Game Over | BPM 逐小节 −3 → 50，所有物种静音 | 4 小节 | 1.0→0 |
| M-08 人类脉动 | 人类圆半径 ≥ 36px | 60Hz sine 低频脉动，与半径正相关 | 即时 | −inf→0.25 |

### 5.2 灾害声层 "4 元素规则"

所有灾害声层必须包含：低频脉动 + 中频颗粒 + 高频闪烁 + 环境音色

| 灾害 | 低频元素 | 中频元素 | 高频元素 | 环境音 |
|------|---------|---------|---------|--------|
| 暴风 storm | 低八度大提琴颤弓 4Hz | 木管颤音 cluster | 镲片 32 分音符 | 风声白噪 + 雷声(12-20s) |
| 野火 wildfire | 大提琴 sul ponticello | 中提琴急促拨弦 | 金属碰撞 + 焦木 | 火焰噼啪 |
| 干旱 drought | 大提琴空弦持续 | 沙锤颗粒 | 钢片琴干涩 | 风沙白噪(高通) |
| 山体滑坡 landslide | 定音鼓心跳(0.5Hz) | 低音木琴下行 | 石头碰撞 | 滚石低通噪声 |
| 栖息地繁盛 flourish | 大提琴泛音列 | 竖琴琶音上行 | 颤音琴+钢片琴双层 | 鸟鸣短句(3段循环) |

## 7. 可生成音频对照表（技能 vs GDD 乐器映射）

> 技能原生支持 7 种乐器：`piano`, `string`, `pad`, `bell`, `bass`, `flute`, `pluck`
> 以下说明 GDD 要求乐器如何在技能中近似实现：

## 附录 A · 生成脚本使用规范

### 背景音乐生成

```bash
# 基本格式
python generate_audio.py "<提示词>" --output-dir ./output

# 带 JSON 输出（用于自动化 pipeline）
python generate_audio.py "<提示词>" --output-dir ./output --json

# 干运行（仅验证提示词有效性）
python generate_audio.py "<提示词>" --output-dir ./output --dry-run
```

### 音效生成

```bash
# UI 音效
python generate_audio.py "UI确认按钮音效，清脆的叮咚声" --output-dir ./output

# 收集音效
python generate_audio.py "物品收集拾取音效，上升的琶音旋律" --output-dir ./output

# 环境循环
python generate_audio.py "洞穴环境音效，低频嗡鸣和混响，15秒" --output-dir ./output
```

### 关键约束

- 所有生成音频为 **纯器乐** — 不含歌词、人声或语音
- 时长范围 0.5–300 秒
- 输出格式 16-bit 44100Hz 立体声 WAV
- 文件名自动包含时间戳防覆盖
- 生成失败时报告错误，不编造路径
